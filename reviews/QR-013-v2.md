## Root Cause Analysis

Excellent root cause identification. The issue was `_find_main_file` returning
`None` when Clang's AST had all user nodes with `includedFrom` set. This caused
the template path (which strips C code) to be used instead of the patched-source
path (which preserves it).

## Three Fixes

### 1. `__main__.py`: source_path fallback
When `collect()` can't determine `source_path` from the AST, falls back to
the `--sources` CLI argument. Uses `Path(src_path).is_file()` to validate.
This ensures the patched-source path is always available for user `.m` files.

### 2. `context.py`: static declarations at original position
Instead of blanking the static declaration and re-emitting in the
`@implementation` preamble, the transpiled declaration is now emitted at
its original file-scope position. This preserves ordering for C functions
that reference the static before `@implementation`.

Key change: `_find_static_var()` helper looks up the `OZStaticVar` to get
the correct C type, then emits `static {c_type} = {init};` at the original
position. The preamble only emits statics not already preserved.

### 3. `emit.py`: orphan statics same logic
`_emit_patched_orphan_source` now checks `orphan_static_names` before
re-emitting, preventing duplication.
