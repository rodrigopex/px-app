## Generated C Quality

The fix changes 5 lines in `_emit_msg_expr` (emit.py:1599-1610):

Before (buggy):
```python
recv_class = _infer_receiver_class(receiver, cls, module) if receiver else cls.name
needs_cast = ret_oz.is_object and recv_class != root_class
if needs_cast:
    out.write(f"(struct {recv_class} *)")
```

After (fixed):
```python
ret_c = ret_oz.c_type
needs_cast = ret_oz.is_object and ret_c != f"struct {root_class} *"
if needs_cast:
    out.write(f"({ret_c})")
```

The root cause was using `_infer_receiver_class` (which returns `__patched__`
when the receiver is an `id`-typed protocol variable) instead of the AST's
declared return type. The fix correctly uses `ret_oz.c_type` — the actual
return type from the method declaration.

This is the right approach: the return type is a property of the method
signature, not the receiver. Clean, minimal fix.
