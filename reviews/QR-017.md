## Generated C Quality

Minimal, correct fix — 5 lines in `_emit_msg_expr` (emit.py):

```python
inferred_class = _infer_receiver_class(...)
defining_class = _find_defining_class(inferred_class, selector, module)
# ...
needs_cast = defining_class != inferred_class
if needs_cast:
    out.write(f"(struct {defining_class} *)")
```

When a subclass calls an inherited method, `inferred_class` is the receiver's
type (e.g., `PXPressureSensor`) and `defining_class` is the ancestor that
implements it (e.g., `PXAnalogSensor`). The cast is safe because C struct
embedding guarantees the parent is at offset 0.

Generated output:
```c
int raw = PXAnalogSensor_readRaw((struct PXAnalogSensor *)self);
```

This is exactly what a human would write.
