## Generated C Quality

The fix is elegant — 6 lines changed in `_emit_return_stmt` (emit.py:1783-1788):

Before (buggy):
```python
out.write(f"{tabs}return ")
_emit_expr(ret_expr, out, ctx)
out.write(";\n")
```

After (fixed):
```python
expr_buf = StringIO()
_emit_expr(ret_expr, expr_buf, ctx)
_flush_pre_stmts(out, ctx, indent)
out.write(f"{tabs}return {expr_buf.getvalue()};\n")
```

The root cause: `_emit_expr` generates `pre_stmts` (like receiver variable
declarations) as side effects in `ctx`, but these were flushed *after* the
`return` keyword was already written. By buffering the expression first and
flushing pre_stmts before writing `return`, the receiver var declaration
now appears in the correct position.

Generated output now correctly produces:
```c
struct OZObject *_oz_recv0 = (struct OZObject *)self->_sensors;
return OZ_SEND_count(_oz_recv0);
```
