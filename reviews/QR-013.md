### OZ-015 (67206d9): Remove `extern` from headers

`_class_header_ctx` in `emit.py:418` — removed the loop that generated
`extern` declarations for static variables. Correct — `static` means
TU-private, exporting via `extern` contradicts the semantics and causes
GCC errors.

## Findings

### Finding 1: [severity: medium] OZ-015 was a regression from OZ-013

OZ-013 added `static` to the `.c` file but didn't remove the `extern`
from the `.h` file. This broke all builds ([URGENT]). The two-commit
fix sequence suggests the OZ-013 change wasn't tested end-to-end before
committing — the unit tests didn't catch the header/source mismatch.

**Recommendation:** Add a test that verifies `static` variables do NOT
appear with `extern` in generated `_ozh.h` headers.
