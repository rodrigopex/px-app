## Context

OZ-002 (bare @synthesize) has two tests — both for bare form only. The fix
changes `_synthesize_properties` to check `existing_ivar_names` before
defaulting to `_propName`. No test verifies that a class with BOTH bare
and explicit @synthesize forms works correctly together.

## Missing edge case

A class with multiple properties:
```objc
@synthesize propA;              /* bare — uses existing ivar */
@synthesize propB = _propB;     /* explicit — creates _propB */
```

Both forms in the same class should resolve correctly without interference.
