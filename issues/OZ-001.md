## Context

Trying to organize source files in subdirectories (`src/services/PXAppConfig.m`)
and import the header from `main.m` via `#import "services/PXAppConfig.h"`.

## Input

```
src/
├── main.m                  ← #import "services/PXAppConfig.h"
└── services/
    ├── PXAppConfig.h
    └── PXAppConfig.m
```

CMakeLists.txt:
```cmake
objz_transpile_sources(app
  src/main.m
  src/services/PXAppConfig.m
)
```

## Observed

The transpiler generates all output files flat into `oz_generated/`:
```
oz_generated/
├── main_ozm.c          ← contains: #include "services/PXAppConfig_ozh.h"
├── PXAppConfig_ozh.h   ← flat, not in services/
└── PXAppConfig_ozm.c
```

GCC error:
```
main_ozm.c:10:10: fatal error: services/PXAppConfig_ozh.h: No such file or directory
```

The transpiler preserves the subdirectory prefix from the `#import` in the
generated `#include` but does not create matching subdirectories in the output,
nor does it flatten the include path.

## Expected

Either:
1. Generate output into matching subdirectories (`oz_generated/services/PXAppConfig_ozh.h`), or
2. Flatten the `#include` path to just `PXAppConfig_ozh.h` since all output is flat

## Workaround

Moved all .m/.h files to flat `src/` layout. See WORKAROUNDS.md entry WA-001.

## Discussion

(empty)

## Resolution

**Root cause:** `_emit_include_replacement()` in `emit.py` extracted the stem from
the full `#import` path (e.g., `"services/PXAppConfig.h"` → `"services/PXAppConfig"`)
preserving the subdirectory prefix. But output files are written flat to `oz_generated/`.

**Fix:** Use `Path(header_name).stem` instead of `header_name.rsplit(".", 1)[0]` to
extract only the filename stem, stripping any directory prefix. This matches the flat
output layout.

**Tests added:**
- `test_include_replacement_flattens_subdir_path` — unit test for the function
- `test_patched_source_flattens_subdir_import` — integration test via patched source

DEV can now revert WA-001 and organize sources in subdirectories.
