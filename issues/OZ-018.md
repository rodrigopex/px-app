## Context

OZ-015 was an [URGENT] regression from OZ-013 — `static` variables were
emitted with `extern` declarations in `_ozh.h` headers, breaking all builds.
The fix (67206d9) removed the extern generation, but no test was added to
prevent this regression from recurring.

This is a high-value regression test: the invariant "file-scope `static`
variables must NEVER appear with `extern` in generated headers" should be
enforced by a test.

## Expected

Add a test that:
1. Creates a class with a file-scope `static` variable
2. Runs `emit()` to generate `_ozh.h` and `_ozm.c`
3. Asserts `extern` does NOT appear in the `_ozh.h` output
4. Asserts `static` DOES appear in the `_ozm.c` output

## Resolution

Added `test_static_var_not_extern_in_header` — verifies `extern` does not
appear for static variables in `_ozh.h` and `static` is present in `_ozm.c`.
