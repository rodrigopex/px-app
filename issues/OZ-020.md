## Context

OZ-003 (object return cast) and OZ-005 (return stmt receiver var) each have
one test covering the basic case. Protocol dispatch is a critical code path
and these fixes touch `_emit_msg_expr` and `_emit_return_stmt` — high-traffic
functions where future changes could regress.

## Missing edge cases

### OZ-003 (protocol dispatch return cast):
- `void`-returning protocol method (should NOT cast)
- `int`-returning protocol method (should NOT cast to struct)
- Nested object return (e.g., method returns `OZArray *` containing `OZString *`)
- Multiple dispatch calls in same method with different return types

### OZ-005 (return stmt receiver var):
- Nested dispatch: `return [[obj array] count];` — two receiver vars needed
- Dispatch in compound expression: `return [a count] + [b count];`
- Dispatch on method result: `return [[self registry] count];`
