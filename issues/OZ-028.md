## Context

OZ-017 (inherited method self cast) has one test covering a 3-level hierarchy
(OZObject → Base → Child). DEV's actual scenario was 5 levels
(OZObject → PXSensorBase → PXAnalogSensor → PXPressureSensor → PXBarometer).
The fix is general but edge cases are untested.

## Missing edge cases

- 4+ level deep inheritance (the exact scenario DEV hit — calling a
  great-grandparent method)
- Multiple inherited method calls in same method body from different
  ancestor levels (e.g., calling both grandparent and parent methods)
- Calling inherited method on a variable (not `self`) that is a subclass type
  (e.g., `[otherObj parentMethod]` where otherObj is declared as ChildClass *)
- Same-class method call (must NOT cast — `defining_class == inferred_class`)
