## Context

The OZ-010 fix updated LIMITATIONS.md to say "switch/case supported with
C enum types". DEV's verification of OZ-007 revealed this is inaccurate —
enum collection works, but switch/case body emission does not (OZ-012).

LIMITATIONS.md should be reverted to indicate switch/case is NOT supported
(only the condition expression is emitted), while noting that enum
declarations ARE now collected.

## Expected

Update LIMITATIONS.md to accurately reflect:
- switch/case is NOT supported (case bodies are not emitted)
- User-defined C enums ARE collected and available for use in if/else chains
- Workaround: use if/else with enum constants

## Resolution

Fixed in same commit as OZ-012 (`5cc5558`). LIMITATIONS.md now accurately states
switch/case is supported. The entry describes enum collection and the requirement
for explicit `enum Name` ivar types.
