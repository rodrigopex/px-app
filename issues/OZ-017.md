## Context

Deep inheritance hierarchy (4+ levels). When a subclass calls an inherited
method via `[self parentMethod]`, the transpiler emits the direct function
call without casting `self` to the parent struct type. GCC 14 treats
`-Wincompatible-pointer-types` as an error.

## Input

```
OZObject → PXSensorBase → PXAnalogSensor → PXPressureSensor → PXBarometer
```

```objc
/* PXPressureSensor.m — level 3, calls level 2 method */
- (int)pressureReading {
    int raw = [self readRaw];           /* readRaw defined in PXAnalogSensor */
    int calibrated = [self calibratedValue]; /* calibratedValue in PXAnalogSensor */
    return calibrated * _pressureScale;
}
```

```objc
/* main.m — calls methods at multiple levels */
PXBarometer *baro = [[PXBarometer alloc] initWithId:99];
int alt = [baro altitude];          /* PXBarometer method */
int pressure = [baro pressureReading]; /* PXPressureSensor method */
int raw = [baro readRaw];           /* PXAnalogSensor method */
int sid = [baro sensorId];          /* PXSensorBase method */
```

## Observed

Generated `PXPressureSensor_ozm.c`:
```c
int PXPressureSensor_pressureReading(struct PXPressureSensor *self)
{
    int raw = PXAnalogSensor_readRaw(self);  /* ERROR: self is PXPressureSensor* */
    int calibrated = PXAnalogSensor_calibratedValue(self);  /* ERROR */
    ...
}
```

Generated `main_ozm.c`:
```c
PXSensorBase_sampleCount(baro);  /* ERROR: baro is PXBarometer* */
PXAnalogSensor_readRaw(baro);    /* ERROR */
```

GCC error:
```
error: passing argument 1 of 'PXAnalogSensor_readRaw' from incompatible
pointer type [-Wincompatible-pointer-types]
note: expected 'struct PXAnalogSensor *' but argument is of type
'struct PXPressureSensor *'
```

## Expected

The transpiler should cast `self` (or the receiver) to the parent struct
type when calling inherited methods:

```c
int raw = PXAnalogSensor_readRaw((struct PXAnalogSensor *)self);
```

This is safe because C struct embedding guarantees the parent struct is at
offset 0 (the `base` field). The generated struct layout is:
```c
struct PXBarometer {
    struct PXPressureSensor base;  /* which embeds PXAnalogSensor, etc. */
    int _altitudeEstimate;
};
```

## Workaround

Cannot workaround in ObjC — the missing casts are in the GENERATED C code
(`PXPressureSensor_ozm.c`), not in user code. Even casting in main.m doesn't
help because the internal `[self parentMethod]` calls also lack casts.

## Discussion

The transpiler uses `_infer_receiver_class()` to determine the receiver type
for method calls. For inherited methods, it should look up the declaring class
(not the receiver's class) and cast accordingly.

All existing samples and tests use at most 2 levels of inheritance
(OZObject → subclass). This is the first test of 4+ levels.

## Resolution

Fixed in `emit.py`: static dispatch path now compares `inferred_class` (receiver
type) with `defining_class` (class that implements the method). When they differ,
a `(struct DefiningClass *)` cast is added to the receiver argument.

Test added: `test_inherited_method_casts_self` — grandparent method call with cast.
