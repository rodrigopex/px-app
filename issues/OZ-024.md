## Context

OZ-007 (user enum collection) has two tests. Enums are now collected from
`.m` files and emitted in class headers. The `_is_user_enum` function filters
system enums — if this filter breaks, system enums could leak into headers
or user enums could be silently dropped.

## Missing edge cases

- Enum with explicit non-sequential values (e.g., 0, 5, 10, 100)
- Enum with negative values
- Enum used as method parameter or return type (not just ivar)
- Multiple enums in same `.m` file
- Enum in `.h` file (must NOT be collected — OZ-016 constraint)
- Anonymous enum (no name — must be skipped)
