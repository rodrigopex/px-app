## Context

Discovered via px-app CHANGELOG Step 11. DEV found that enum declarations
in `.h` files are not collected by the transpiler — only enums in `.m` files
processed by `objz_transpile_sources` are collected. This is a significant
constraint for users who expect to share enum types across multiple classes
via a header file.

LIMITATIONS.md's switch/case entry says "user-defined C enums from the main
source file are collected" but doesn't explicitly state the `.m`-only
restriction.

## Expected

Update LIMITATIONS.md switch/case or enum entry to explicitly state:
- Enum declarations must be in `.m` files (not `.h` files)
- Only `.m` files listed in `objz_transpile_sources()` are scanned
- To share an enum across classes, define it in each `.m` file or
  use `static int` constants in a shared `.h`

Updated LIMITATIONS.md switch/case entry to explicitly note that only enums in
