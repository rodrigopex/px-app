## Context

When a `.m` file uses `#include "my_c_header.h"` to include a plain C header
(containing C structs, function declarations, macros), the transpiler strips
the directive from the generated output. This prevents C interop from working
directly in `.m` files.

Angle-bracket includes (`#include <header.h>`) ARE preserved. But quoted
includes are treated as ObjC headers and either transformed to `_ozh.h`
references or dropped entirely.

## Input

```objc
/* PXZbusPublisher.m */
#import "PXZbusPublisher.h"
#include "px_zbus_defs.h"   /* plain C header with struct + function decls */
#include <zephyr/kernel.h>

@implementation PXZbusPublisher
- (int)publish:(int)value {
    struct px_sensor_msg msg;       /* from px_zbus_defs.h */
    msg.value = value;
    return px_zbus_publish(&msg);   /* C function from px_zbus_defs.h */
}
@end
```

## Observed

Generated C:
```c
#include "PXZbusPublisher_ozh.h"
/* ... auto-generated includes ... */
#include <zephyr/kernel.h>      /* ← preserved (angle brackets) */
/* #include "px_zbus_defs.h"    ← STRIPPED */
```

GCC errors:
```
error: storage size of 'msg' isn't known
error: 'px_zbus_publish' undeclared
```

The transpiler also strips:
- `ZBUS_CHAN_DEFINE(...)` macro calls (C macros not collected)
- Plain C function declarations (`int foo(int x);`)
- C struct definitions (when defined in the `.m` file)

## Expected

`#include "non_objc_header.h"` should be preserved in the generated output
when the included file is NOT an ObjC header (no `@interface`, `@protocol`,
etc.). The transpiler already detects ObjC headers via `_is_objc_header()` —
non-ObjC headers should pass through unchanged.

## Workaround

Split C code into separate `.c` file compiled directly by CMake:
1. Put C macros/structs/functions in `px_zbus.c` (via `target_sources`)
2. Put declarations in `px_zbus_defs.h`
3. Include via angle brackets: `#include <px_zbus_defs.h>` (add dir to
   `target_include_directories`)
4. ObjC class calls C functions, main.m bridges via angle-bracket include

See WORKAROUNDS.md entry WA-005.

## Resolution

MAINTAINER could not reproduce. Both template and patched source paths correctly
preserve `#include "plain_c_header.h"` when the header does not contain ObjC
syntax (`@interface`, `@implementation`).

Verified:
- `_is_user_include()` returns True for non-ObjC quoted includes
- `_emit_include_replacement()` passes through non-ObjC includes unchanged
- `_emit_patched_source()` dedup only strips `_ozh.h` includes
- End-to-end test confirms `#include "px_zbus_defs.h"` appears in output

Possible DEV-side causes:
1. Header file contains `@interface` or `@implementation` in a comment/string
2. Header file is not found at the expected relative path
3. DEV tested before OZ-007/OZ-012 fixes (which improved roundtrip)

DEV should retry with latest `west update` and verify the header file content.
