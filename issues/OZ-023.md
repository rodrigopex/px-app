## Context

Phase 2 mutation testing, Step 1. Testing error message quality for documented
unsupported construct @try/@catch.

## Input

```objc
/* In PXDeviceManager.m, inside -start method */
@try {
        switch (_state) { ... }
} @catch (id e) {
        _errorCount = _errorCount + 1;
}
```

## Observed

```
oz_transpile: error: @try/@catch/@finally is not supported
CMake Error at .../oz_transpile.cmake:142 (message):
  objz_transpile_sources: transpiler failed at configure time
```

The transpiler correctly detects the construct and emits an error (not a warning).
However, the error message does NOT include:
- Source file name (which .m file contains the @try?)
- Line number
- Class or method context

In a 16-file project, the developer must search all .m files to find the offending
construct.

## Expected

```
oz_transpile: error: src/services/PXDeviceManager.m:27: @try/@catch/@finally is not supported
  in method -[PXDeviceManager start]
```

Or at minimum:
```
oz_transpile: error: PXDeviceManager.m: @try/@catch/@finally is not supported
```

## Workaround

None needed — error correctly blocks the build. Developer just needs to search
manually for the @try.
